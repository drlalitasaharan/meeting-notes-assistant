# External demo checklist

## Environment

- [ ] On latest `main`
- [ ] Docker stack starts successfully
- [ ] `/healthz` returns status `ok`
- [ ] Backend is reachable at `localhost:8000`
- [ ] Worker is running
- [ ] Demo audio file is available
- [ ] Validated backup artifact is available

## Demo artifacts

- [ ] Client demo script reviewed
- [ ] Presenter runbook reviewed
- [ ] Fallback plan reviewed
- [ ] Validated 10-minute markdown opened once before call
- [ ] Browser tabs prepared
- [ ] Terminal prepared
- [ ] No private or irrelevant local files visible

## Product output checks

- [ ] Purpose section is readable
- [ ] Outcome section is readable
- [ ] Key Points are useful
- [ ] Decisions are visible
- [ ] Action Items are visible
- [ ] No malformed phrase appears
- [ ] No extra spaces before periods appear

## Messaging

- [ ] Position as controlled external pilot/demo ready
- [ ] Do not claim broad production readiness
- [ ] Emphasize structured notes over raw transcript
- [ ] Emphasize decisions and action items
- [ ] Close with pilot next step

## Final go/no-go

Go if:

- The stack is healthy
- The backup artifact is ready
- The presenter can explain the flow clearly
- The output is clean enough to show externally

No-go if:

- Backend cannot start
- Backup artifact is missing
- Output contains obvious publishability issues
