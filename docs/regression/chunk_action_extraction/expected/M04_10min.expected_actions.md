## Regression purpose

This file closes the public-launch 10-minute regression coverage gap.

M04 is a decisions-focused medium meeting sample. It is included to verify that the action-recall pipeline does not invent vague or non-committal action items when the meeting contains decisions and discussion but limited explicit assigned work.

## Expected action behavior

Expected action item count: 0–1

Acceptable output:
- No action items, if no explicit owner/task/deadline is present.
- At most one clearly grounded action item, only if the transcript contains a specific owner and concrete deliverable.

Unacceptable output:
- Vague action items.
- Non-committal actions such as “do something with...”
- Decision statements converted into fake action items.
- Generic “review summary,” “copy/export,” or workflow-instruction actions.

## Pass condition

PASS if MeetIQ preserves decisions/summary content while avoiding fake action items.

FAIL if the output creates vague, generic, or unsupported action items.
